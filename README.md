## Instalación :arrow_down:

### Requisitos previos

Antes de comenzar, asegúrate de tener instalado lo siguiente:

| Herramienta             | Versión mínima | Enlace de descarga                           |
| ---------------------- | --------------- | ------------------------------------------- |
| **Node.js**            | v18.17.0+       | [Descargar](https://nodejs.org/en/download/) |
| **Git**                | Última          | [Descargar](https://git-scm.com/downloads)   |
| **pnpm** (recomendado) | Última          | [Instalar](https://pnpm.io/installation)     |

> **Nota**: Next.js 16 requiere Node.js 18.17 o superior. Se recomienda Node.js 20+ para un rendimiento óptimo.

#### Verificar instalación

Comprueba tus instalaciones con estos comandos:

```bash
node --version
git --version
pnpm --version  # o npm --version
```

## Primeros pasos :dart:

### 1. Fork y clona el repositorio

```bash
git clone https://github.com/<YOUR_GITHUB_USERNAME>/developer-portfolio.git
cd developer-portfolio
```

### 2. Instala dependencias

```bash
# Usando pnpm (recomendado)
pnpm install

# Usando npm
npm install

# Usando yarn
yarn install
```

### 3. Configura las variables de entorno

```bash
cp .env.example .env
```

Edita el archivo `.env` con tus valores (ver la sección [Uso](#usage-joystick)).

### 4. Ejecuta el servidor de desarrollo

```bash
pnpm dev
# o
npm run dev
# o
yarn dev
```

Abre [http://localhost:3000](http://localhost:3000) en tu navegador.

---

### 🐳 Despliegue con Docker (Alternativa)

#### Opción 1: Usar Docker Compose (Recomendado)

```bash
docker-compose up --build

# Ejecutar en modo desapegado
docker-compose up -d --build

# Detener
docker-compose down
```

#### Opción 2: Usar Docker directamente

**Para desarrollo:**

```bash
# Construir la imagen de desarrollo
docker build -t developer-portfolio:dev -f Dockerfile.dev .

# Ejecutar el contenedor
docker run -p 3000:3000 --name portfolio-dev developer-portfolio:dev

# Detener y remover el contenedor
docker stop portfolio-dev && docker rm portfolio-dev
```

**Para producción:**

```bash
# Construir la imagen de producción
docker build -t developer-portfolio:prod -f Dockerfile.prod .

# Ejecutar el contenedor de producción
docker run -p 3000:3000 --name portfolio-prod developer-portfolio:prod
```

## Uso :joystick:

### Configuración de variables de entorno

Crea un archivo `.env` en la raíz con las siguientes variables:

```env
# Google Tag Manager (Opcional - para analytics)
NEXT_PUBLIC_GTM=GTM-XXXXXXX

# URL pública de tu aplicación
NEXT_PUBLIC_APP_URL=https://tu-dominio.com

# Configuración de Telegram para notificaciones del formulario de contacto
TELEGRAM_BOT_TOKEN=tu_token_de_bot_aqui
TELEGRAM_CHAT_ID=tu_chat_id_aqui

# Configuración de Gmail para correos del formulario de contacto
GMAIL_PASSKEY=tu_contraseña_de_app_gmail
EMAIL_ADDRESS=tu_email@gmail.com
```

#### Descripción de variables:

| Variable              | Obligatoria | Descripción                                  |
| --------------------- | ----------- | -------------------------------------------- |
| `NEXT_PUBLIC_GTM`     | No          | ID de Google Tag Manager para analíticas     |
| `NEXT_PUBLIC_APP_URL` | Sí          | URL pública de tu portafolio                 |
| `TELEGRAM_BOT_TOKEN`  | No          | Token del bot de Telegram para notificaciones |
| `TELEGRAM_CHAT_ID`    | No          | ID del chat de Telegram para recibir mensajes |
| `GMAIL_PASSKEY`       | No          | Contraseña de app de Gmail para notificaciones de email |
| `EMAIL_ADDRESS`       | No          | Tu correo de Gmail para enviar mensajes      |

> **Nota**: Las funciones del formulario de contacto requieren al menos Telegram o Gmail configurados (o ambos).

### Personaliza los datos de tu portafolio

Todo el contenido del portafolio se administra mediante archivos de datos en la carpeta `utils/data/`:

#### 📝 Información personal (`personal-data.js`)

```javascript
export const personalData = {
  name: "YOUR NAME",
  profile: "/profile.png", // Ruta de tu imagen de perfil
  designation: "Software Developer", // Tu título profesional
  description: "Your bio and introduction...", // Sobre ti
  email: "your.email@example.com",
  phone: "+1234567890",
  address: "City, Country",
  github: "https://github.com/yourusername",
  facebook: "https://www.facebook.com/yourprofile",
  linkedIn: "https://www.linkedin.com/in/yourprofile",
  twitter: "https://twitter.com/yourusername",
  stackOverflow: "https://stackoverflow.com/users/your-id",
  leetcode: "https://leetcode.com/yourusername/",
  devUsername: "yourusername", // Usuario de dev.to para integración de blog
  resume: "https://link-to-your-resume.pdf",
};
```

#### 💼 Archivos de datos adicionales

| Archivo             | Propósito                                         |
| ------------------ | ------------------------------------------------- |
| `experience.js`    | Tu experiencia laboral e historial profesional    |
| `projects-data.js` | Proyectos del portafolio con descripciones y enlaces |
| `skills.js`        | Habilidades técnicas y competencias               |
| `educations.js`    | Formación académica y certificaciones             |
| `contactsData.js`  | Configuración del formulario de contacto          |

#### 🎨 Agregar tu imagen de perfil

Coloca tu foto en el directorio `public/` y actualiza el campo `profile` en `personal-data.js`:

```javascript
profile: "/tu-nombre-de-imagen.png"; // o .jpg, .webp
```

## Tutoriales :wrench:

### 📧 Configuración de contraseña de app de Gmail

1. Ve a [https://myaccount.google.com/](https://myaccount.google.com/)
2. Navega a **Seguridad** → **Verificación en dos pasos** (activa si no está habilitada)
3. Ve a **Seguridad** → **Contraseñas de apps**
4. Selecciona app: **Correo**, dispositivo: **Otro (nombre personalizado)**
5. Genera y copia la contraseña de 16 caracteres
6. Añade en el archivo `.env`:

```env
GMAIL_PASSKEY=abcd efgh ijkl mnop
EMAIL_ADDRESS=tu.email@gmail.com
```

---

### 🤖 Crear un bot de Telegram

1. Abre Telegram y busca `@BotFather`
2. Envía el comando `/newbot`
3. Establece el nombre y el nombre de usuario del bot (debe terminar en `bot`)
4. Copia el token del bot
5. Envía un mensaje a tu bot
6. Obtén el chat ID desde: `https://api.telegram.org/bot<BOT_TOKEN>/getUpdates`
7. Añade en el archivo `.env`:

```env
TELEGRAM_BOT_TOKEN=123456789:ABCdefGHIjklMNOpqrsTUVwxyz
TELEGRAM_CHAT_ID=123456789
```

---

### 📝 Obtener blog desde dev.to

1. Crea una cuenta en [dev.to](https://dev.to/)
2. Abre `utils/data/personal-data.js`
3. Establece tu nombre de usuario de dev.to:

```javascript
export const personalData = {
  // ... otros campos
  devUsername: "yourusername",
};
```

El portafolio obtiene y muestra automáticamente tus artículos públicos más recientes. No se requiere clave de API.

## Resolución de problemas :wrench:

### Problemas comunes y soluciones

<details>
<summary><strong>❌ "next no se reconoce como un comando interno o externo"</strong></summary>

**Solución:**

```bash
# Opción 1: Instalar Next.js globalmente
npm install -g next

# Opción 2: Usar npx (recomendado)
npx next dev

# Opción 3: Usar los scripts del gestor de paquetes
npm run dev
```

</details>

<details>
<summary><strong>❌ El puerto 3000 ya está en uso</strong></summary>

**Solución:**

```bash
# Encuentra y mata el proceso que usa el puerto 3000
# En macOS/Linux:
lsof -ti:3000 | xargs kill -9

# En Windows:
netstat -ano | findstr :3000
taskkill /PID <PID> /F

# O usa otro puerto:
PORT=3001 npm run dev
```

</details>

<details>
<summary><strong>❌ Módulo no encontrado o errores de dependencias</strong></summary>

**Solución:**

```bash
# Elimina la caché y reinstala dependencias
rm -rf node_modules package-lock.json
npm cache clean --force
npm install

# O con pnpm:
rm -rf node_modules pnpm-lock.yaml
pnpm store prune
pnpm install
```

</details>

<details>
<summary><strong>❌ Las variables de entorno no funcionan</strong></summary>

**Solución:**

- Asegúrate de que el archivo `.env` esté en la raíz del proyecto
- Reinicia el servidor de desarrollo después de cambiar `.env`
- Verifica que las variables que comienzan con `NEXT_PUBLIC_` se usan en el código del cliente
- Las variables del lado del servidor NO deben comenzar con `NEXT_PUBLIC_`

</details>

<details>
<summary><strong>❌ Las imágenes no se cargan</strong></summary>

**Solución:**

- Verifica que las imágenes estén en la carpeta `public/`
- Usa rutas que comiencen con `/` (por ejemplo, `/profile.png`)
- Comprueba que las extensiones de archivo coincidan con el código
- Asegúrate de que las imágenes estén comprometidas en el repositorio

</details>

<details>
<summary><strong>❌ El formulario de contacto no envía correos</strong></summary>

**Solución:**

- Verifica que la contraseña de app de Gmail sea correcta (16 caracteres)
- Comprueba que la verificación en dos pasos esté habilitada en tu cuenta de Google
- Asegúrate de que `EMAIL_ADDRESS` coincida con la cuenta de Gmail
- Prueba el token de bot de Telegram y el chat ID por separado
- Revisa la consola del navegador para mensajes de error

</details>
